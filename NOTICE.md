Supplementary terms to [LICENSE](LICENSE).

1. **Fork Network and Substantial Changes**
   - Software and derivatives must remain within the original fork network unless **substantial changes** are made.  
   - **Substantial changes** include:

     - Meaningful change in purpose, domain, or design.  
   - Minor edits (docs, bug fixes, formatting) are **not** substantial.

2. **Author and Commit History**
   - Original author(s) must remain credited in all commit histories.
   - Existing commit metadata must **not** be altered, removed, or replaced, except for legitimate privacy requests.

3. **Repository Name and Attribution**
   - Projects leaving the fork network **cannot use the original repository or project name**.  
   - **Forks inside the network:** must provide **visible credit** to the original repository and author(s) (e.g., README, project description).  
   - **Projects outside the network:** must provide **accessible credit** to the original repository and author(s), but it need not be as prominently displayed.

4. **Credit**
   - Original author(s) must always be credited as contributors in derivatives, forks, or redistributions.
   - Credit can appear in documentation, acknowledgments, source headers, or other accessible locations as appropriate.
